## Fluxo Técnico — Como Funciona Agora

### `/generate` (Kling single)
1. Usuário sobe vídeo de referência + imagem do avatar
2. Clica "Gerar" → `POST /api/generate-kling` → cria task na kie.ai direto
3. Redireciona para `/jobs/[id]`
4. `JobStatusPoller` chama `GET /api/jobs/[id]` a cada 3s
5. `/api/jobs/[id]` detecta `provider=kling` + `externalTaskId` → faz polling kie.ai → atualiza DB
6. Quando concluído, mostra o vídeo

### `/takes` (Kling multi-take)
1. Usuário sobe N pares (imagem + vídeo) — cada par é um take
2. Clica "Gerar" → `POST /api/generate-kling` para cada take em paralelo
3. Polling individual via `GET /api/jobs/[jobId]` a cada 3s
4. Cada job `/api/jobs/[id]` faz polling kie.ai automaticamente
5. Quando todos concluem, mostra grade de vídeos (merge era via `/api/merge` que retorna 503 — não funciona)

### `/animate` (SeedDance multi-take)
1. Usuário sobe N fotos — cada uma com sua própria descrição
2. Clica "Gerar Prompts" → `POST /api/generate-prompt` em paralelo para cada take
3. Revisa/edita prompts gerados
4. Clica "Animar" → `POST /api/animate` para cada take em paralelo
5. Polling via `GET /api/animate/[id]` a cada 3s por take
6. Quando ≥2 concluem → aparece "Quer juntar em um só?"
7. Merge client-side via `mergeVideosClient()` → download

### Merge de Vídeos (`/join` e inline no `/animate`)
- **Biblioteca:** nenhuma — usa Canvas + MediaRecorder (APIs nativas do browser)
- **Arquivo:** `apps/web/lib/merge-videos.ts` → `mergeVideosClient(files, onProgress)`
- **Tempo:** igual à duração total dos vídeos (processamento em tempo real)
- **Formato:** MP4 se suportado, senão WebM
