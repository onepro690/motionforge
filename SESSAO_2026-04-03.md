# Sessão 2026-04-03 — MotionForge

## O que foi feito nessa sessão

### 1. Corrigido: Jobs ficavam presos em QUEUED no Vercel
**Problema:** BullMQ worker só roda localmente. No Vercel, jobs ficavam em "Na fila — aguardando processamento..." para sempre.

**Solução:** Criadas rotas dedicadas que chamam a kie.ai diretamente, sem fila:
- `POST /api/animate` → cria task SeedDance direto na kie.ai, salva `externalTaskId` no banco
- `GET /api/animate/[id]` → consulta status da task na kie.ai a cada 3s, atualiza DB

### 2. Corrigido: Model ID errado do SeedDance
**Problema:** Model `"seeddance"` retornava 422 da kie.ai.
**Solução:** Model correto é `bytedance/seedance-1.5-pro`

### 3. Corrigido: Parâmetro `duration` deve ser string
**Problema:** kie.ai retornava 500 `"duration it must be a string"`.
**Solução:** `duration` deve ser `"4"`, `"8"` ou `"12"` (string, não número). Selector ajustado para só mostrar esses valores.

### 4. Novo: Página "Animar por Texto" (`/animate`) — Multi-take com prompts individuais
- Cada foto (take) tem sua própria descrição e prompt
- Prompts gerados via GPT-4o Mini em paralelo para todos os takes
- Todos os takes enviados ao SeedDance em paralelo
- Polling individual por take com barra de progresso
- Ao concluir ≥2 takes: aparece "Quer juntar esses vídeos em um só?"
- Junção automática com download do arquivo final

### 5. Novo: Página "Juntar Vídeos" (`/join`)
- Upload de múltiplos vídeos com reordenação (setas)
- Merge client-side usando **Canvas + MediaRecorder** (sem ffmpeg, sem dependências externas)
- Funciona direto no browser, sem servidor
- Tempo de processamento = duração total dos vídeos

### 6. Utilitário compartilhado
- `apps/web/lib/merge-videos.ts` → `mergeVideosClient(files, onProgress)` — usado tanto em `/join` quanto em `/animate`

### ⚠️ Pendente / Bugs Conhecidos

#### Crítico: `/generate` e `/takes` ficam QUEUED no Vercel
**Causa:** Usam `/api/jobs` → BullMQ → Worker não roda no Vercel
**Solução:** Criar rotas diretas para Kling similar ao que foi feito para SeedDance:
- `POST /api/generate-kling` — cria task Kling direto na kie.ai
- `GET /api/generate-kling/[id]` — polling direto na kie.ai

A lógica é idêntica ao `/api/animate`, mas usando a API do Kling (`kling-3.0/motion-control`) em vez do SeedDance.

#### Kling API (para implementar amanhã)
```
POST https://api.kie.ai/api/v1/jobs/createTask
{
  "model": "kling-3.0/motion-control",
  "input": {
    "image_url": "<url>",
    "video_url": "<url>",
    // outros parâmetros do Kling
  }
}
```
